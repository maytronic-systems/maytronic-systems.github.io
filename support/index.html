FAQ

Why does IRIS ask for usage-related access?

Some IRIS views depend on Android system APIs that require explicit access from the user before app activity summaries can be displayed.

Why can traffic values differ from carrier or router measurements?

Different systems use different sampling intervals, boundaries, and aggregation rules, so totals can vary across sources.

Why is some data incomplete for certain apps?

Android system limitations, OEM differences, and short collection history can reduce the amount of information available for some apps.

Does IRIS block other apps or network connections?

No. IRIS is designed as an inspection and visualization product, not as a blocking or intervention tool.

Troubleshooting checklist

  1. Update IRIS to the latest available version.
  2. Restart the device if system-provided statistics appear stale.
  3. Confirm that any required Android access has been granted.
  4. Review battery optimization settings if background behavior is relevant.
  5. Send a bug report with reproduction steps if the issue persists.
